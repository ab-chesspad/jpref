JPref V 0.0.0.1

The goal of this project is building a program to play Russian Preferans, two bots agains a human player. Honest game, no peeping or cheating.
This version is far from completion. It's only a graphical ptatform for testing.
It can only do bidding and play all-pass.

Installation
The program can run on all OS that support standard Java, like Windows, macOS, Linux.
JRE 1.8 or later is needed to run JPref.
If you do not have JRE installed, you can find all the necessary instructions here: www.java.com/en/download/help/download_options.html. If you do not plan to build this project and just want to play with the program, JRE will be enough, JDK is not necessary. In this case you can get the pre-build program here.

Launch
Open Terminal window (cmd.exe on Mindows) and type:
java -jar <path to jpref.jar> [list of deals]
If the list of deals is specified, it will be used for play instead of random deals. This allows better program testing.
One can use a log that the program generates (see below) as a list of deals.
JPref can work на русском и in English. If you use English OS and want JPref to use Russian, change the launching command to:
java -Duser.language=ru -jar <path to jpref.jar> [list of deals]

Running the program
JPref can only play all-pass, so this version goes directly there, skipping bidding. Also card selection during play is simplified, it requires a single left-button mouse click (in the real game the confirmation will be needed).
The program menu is invoked with a right-button mouse click:
  • New Game - the current gane will be aborted.
  • Replay - replay the current deal.
  • Пуля - not implemented yet.
  • Послать лог - see below.
The program has a bunch of settings, but so far only the window location and size can be changed, the rest are not configurable.
The important program feature is logging. The program records there all the games. Logs are located in 'logs' directory, either next to jpref.jar file (macOS, Linux), or in <your home folder>\jpref (Windows). New log is opened with a new program launch or with e new day.
Button Comments (second right in the bottom right corner) invokes a comment window. Important to record program errors, stupid moves, etc.
The whole comment text will be written into the log that the user can submit to the developer:
  • Through the menu. Limitations - only the current log and once a day.
  • Via github - open an issue.
  • Send the log file to ab.jpref@gmail.com

Exit
It is recommended to exit the program via 'x' button at the top of the program window. Then it will remember the window size and location and it will open it on the same spot next time.
